Privacy Policy

Usage of Data

ZeroTwo, the bot, may use stored data, as defined below, for different features including but not limited to: welcoming joining users (when enabled) and handling commands.
No usage of data outside of the mentioned cases will happen and the data is not shared with any third-party service.

Stored Data

ZeroTwo may store the following data automatically as soon as being added to a new Discord server:
• id - the server's ID
• name - the server's name
• owner - the server's owner information (e.g name, tag, id)
• region - the server's preferred region
• joinedAt - the exact time the bot joined the server
• language - the language ZeroTwo responds in (default en) • prefix - the prefix to execute commands (default zt-)
• blacklisted - whether the server is blacklisted or not (defaults to false
• welcome - whether the bot welcomes new users or not (defaults to false)
• level - whether the bot's leveling module is enabled or not (defaults to false)
• leveling_up_message - the message to send when leveling up (defaults to {member}, congratulations! You have leveled up to **{member_level}**! :tada:)
• counting - whether the bot's counting module is enabled or not (defaults to false)
• invite - whether the bot's invite tracker module is enabled or not (defaults to false)
• notifications - whether the bot's social media notifications module is enabled or not (defaults to false)
• anti_phishing - whether the bot's anti-phishing module is enabled or not (defaults to true)
• automoderation - whether the bot's automoderation module is enabled or not (defaults to false)

No other data than the ones mentioned above will be stored upon joining a server.

Updating Data

Users may choose to update specific data by using ZeroTwo's commands.
Updating data may require input by an end-user, certain data can be seen as mature/sensitive, and thus may be restricted and/or hidden from public view.

No other actions may update data besides the ones mentioned above.

Temporarily Stored Data

ZeroTwo may keep stored data inside an internal cache for a certain amount of time.
After this time period, the cached information will be removed and only be re-added when required.
Data may be removed from cache pre-maturely through actions such as removing the bot from the corresponding server.

Removal of Data

Automatic Removal of Data

Stored Data can be removed automatically through means of removing ZeroTwo from a server.
Re-inviting ZeroTwo will add the same default values, as mentioned above, back to the bot's database.

Manual Removal of Data

Manual removal of stored data can be requested through email at legal@zerotwo.wtf
For security reasons, we will ask you to provide us with proof of ownership of the server that you wish the data to be removed from.
Manual removal of Data can only be requested by the server owner.
If ZeroTwo is still present after the data removal, ZeroTwo will be manually removed by a bot-administrator.

Requesting a Copy of Data

A copy of the stored data can be requested through email at legal@zerotwo.wtf
For security reasons, we will ask you to provide us with proof of ownership of the server that you wish the data to be copied and sent to you from.

Questions or Concerns

If you have any questions or concerns regarding the privacy policy, please contact us at legal@zerotwo.wtf or through our Discord server.
